# skill-sources

Curated source library for [Slideless](https://github.com/gogoshaka/slideless) deck research.

URLs are organized by topic, pushed by a browser extension, and consumed by the Slideless researcher agent. Tags are auto-generated by GitHub Models.

## How it works

```
Browser Extension → this repo → GitHub Action → Researcher Agent → Deck Author
     (push URL +     (store)     (enrich tags     (read + fetch)     (build slides)
      page excerpt)               via LLM)
```

1. **Browse** — find a useful article, blog post, or doc
2. **Save** — click the browser extension to push the URL with topic, priority, and page excerpt
3. **Enrich** — a GitHub Action calls GitHub Models to auto-generate tags from the excerpt
4. **Research** — the Slideless researcher agent reads curated URLs to seed deck research
5. **Build** — the deck author uses extracted claims to build slides

## Source priority

| Priority | Meaning | Examples |
|----------|---------|----------|
| **P0** | Official first-party | Vendor blogs, official docs, vendor webinars |
| **P1** | Reputable third-party | Conference blogs, industry analysts, tech publications |
| **P2** | Community content | Forum threads, personal blogs, unverified posts |

## Auto tag generation

When a source is saved, the extension extracts a page excerpt (title, meta tags, headings, body text) using `chrome.scripting.executeScript` on the active tab — **no content script, no broad permissions**.

Tags are generated through a two-tier strategy:

### Tier 1 — Client-side (Prompt API, instant)

If the browser supports the [Prompt API](https://developer.chrome.com/docs/ai/prompt-api) (Chrome/Edge 138+, Gemini Nano on-device), tags are generated **locally** before saving:

- No network call, no latency, no `_excerpt` stored in the repo
- Runs on-device via Gemini Nano — private and offline-capable
- Requires 22GB disk space for the model (downloaded once)
- Falls back to Tier 2 automatically if unavailable

### Tier 2 — Server-side (GitHub Action, fallback)

If the Prompt API isn't available, the excerpt is stored as a temporary `_excerpt` field. A GitHub Action then:

1. Detects sources with `_excerpt`
2. Calls **GitHub Models** (`gpt-5.4-mini`) to generate tags
3. Merges generated tags with any manual tags (deduped, capped at 15)
4. Removes `_excerpt` and commits the enriched source

**Parameters:** `temperature: 0.3`, `max_tokens: 200`

### Shared prompt

Both tiers use the same prompt for consistency:

```
You are a tag generator for a technical knowledge base.

Given the title and content excerpt of a web page, generate 3-8 concise,
lowercase, kebab-case tags that capture the key topics, technologies, and concepts.

Rules:
- Tags must be lowercase kebab-case (e.g., "incident-graph", "microsoft-sentinel", "kql")
- Focus on technologies, products, concepts, and techniques
- Avoid generic tags like "blog", "article", "security" unless highly specific
- Prefer specific terms over broad ones
- Return ONLY a JSON array of strings, nothing else
```

## Browser extension

### Install

1. Clone this repo or download the `extension/` folder
2. Open `chrome://extensions` → enable **Developer mode**
3. Click **Load unpacked** → select the `extension/` folder

### Setup

When prompted, enter:
- **Client ID:** `Ov23liqQtNWKD59WA08S`
- **Repository:** `gogoshaka/skill-sources`

### Authentication

The extension uses **GitHub OAuth Device Flow** — no backend server needed:
1. Click **Connect to GitHub**
2. A code appears (e.g., `A1B2-C3D4`)
3. Open `github.com/login/device` and enter the code
4. The extension polls in the background and stores the token

Only the `public_repo` scope is requested.

### Permissions

| Permission | Why |
|------------|-----|
| `activeTab` | Read URL and title of the current tab when you click the extension |
| `scripting` | Extract page content (headings, meta, body) for tag generation |
| `storage` | Store GitHub token and settings across sessions |
| `https://api.github.com/*` | Push sources to this repo via GitHub Contents API |
| `https://github.com/*` | OAuth Device Flow authentication |

**No content script.** Content extraction only runs when you click the extension, on the current tab only.

## Contributing

### Via browser extension (recommended)

Click the extension on any page → select topic → Save.

### Manually

1. Edit or create a topic file in `topics/`
2. Add your source entry following the schema
3. Open a PR

All sources must include: `url`, `title`, `priority`, `summary`, `addedAt`, `addedBy`.
